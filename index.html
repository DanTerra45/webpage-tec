Element Description Example More information
HTML Wraps all the content on the entire page and is sometimes known as the root element. It also includes the lang attribute, setting the primary language of the document. <html lang="your-lang-type-here"></html> Link
Input Creates interactive controls for web-based forms in order to accept data from the user. <input type="text" id="name" name="name"/> Link
Header Represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements. <header> <h1>One big title here</h1> ... you can combine with more elements like H2, H3, H6, time, you name it! </header> Link
Footer Typically contains information about the author of the section, copyright data or links to related documents. <footer> <p>© 2023 Panfleto Jamaicano</p> </footer> Link
Article Represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication). <article> <h2>13 September 2023</h2> <p>Possibly Rain.</p> </article> Link
Section Represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it. <section> <h2>Introduction</h2> <p>This document provides a guide to help with the important task of choosing the correct Apple.</p> </section> Link
Canvas Allows to draw graphics and animations on screen. <canvas width="120" height="120"> A text to describe what is this canvas. </canvas> Link
Audio Allows us to embed sound content in documents. It may contain one or more audio sources, and that can be represented by using the src attribute or the <source>, src element. <audio controls src="/root/aud/beep.flac"> We can use elements like figcaption to describe what is this sound (the upper text) before audio tag. </audio> Link
Body Represents the content of an HTML document. There can be only one <body> element in a document. <body> ...(anything possible can go here)... </body> Link
Head Contains machine-readable information (metadata) about the document, like its title, scripts, and style sheets. Like, for example this document! <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <link rel="stylesheet" href="styles.css"> <title>HTML Elements</title> </head> Link
