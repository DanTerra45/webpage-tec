HTML Wraps all the content on the entire page and is sometimes known as the root element. It also includes the lang attribute, setting the primary language of the document. <html lang="your-lang-type-here"></html> Link
Input Creates interactive controls for web-based forms in order to accept data from the user. <input type="text" id="name" name="name"/> Link
Header Represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements. <header> <h1>One big title here</h1> ... you can combine with more elements like H2, H3, H6, time, you name it! </header> Link
Footer Typically contains information about the author of the section, copyright data or links to related documents. <footer> <p>Â© 2023 Panfleto Jamaicano</p> </footer> Link
Article Represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication). <article> <h2>13 September 2023</h2> <p>Possibly Rain.</p> </article> Link
Section Represents a generic standalone section of a document, which doesn't have a more specific semantic element to represent it. <section> <h2>Introduction</h2> <p>This document provides a guide to help with the important task of choosing the correct Apple.</p> </section> Link
Canvas Allows to draw graphics and animations on screen. <canvas width="120" height="120"> A text to describe what is this canvas. </canvas> Link
Audio Allows us to embed sound content in documents. It may contain one or more audio sources, and that can be represented by using the src attribute or the <source>, src element. <audio controls src="/root/aud/beep.flac"> We can use elements like figcaption to describe what is this sound (the upper text) before audio tag. </audio> Link
Body Represents the content of an HTML document. There can be only one <body> element in a document. <body> ...(anything possible can go here)... </body> Link
Head Contains machine-readable information (metadata) about the document, like its title, scripts, and style sheets. Like, for example this document! <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <link rel="stylesheet" href="styles.css"> <title>HTML Elements</title> </head> Link
Abbr Provides a full expansion of the term in plain text on first use, along with the <abbr> to mark up the abbreviation. <abbr> </abbr> Link
Address Indicates that the enclosed HTML provides contact information for a person or people, or for an organization. <address> </address> Link
Area Defines an area inside an image map that has predefined clickable areas. An image map allows geometric areas on an image to be associated with hypertext links. <area> </area> Link
Aside Represents a portion of a document whose content is only indirectly related to the document's main content <aside> </aside> Link
Base Specifies the base URL to use for all relative URLs in a document <base> </base> Link
Bdi Tells the browser's bidirectional algorithm to treat the text it contains in isolation from its surrounding text. <bdi> </bdi> Link
Bdo Overrides the current directionality of text, so that the text within is rendered in a different direction. <bdo> </bdo> Link
Blockquote Indicates that the enclosed text is an extended quotation. Usually, this is rendered visually by indentation. <blockquote> </blockquote> Link
Br Produces a line break in text (carriage-return). It is useful for writing a poem or an address, where the division of lines is significant. <br> </br> Link
Button Interactive element activated by a user with a mouse, keyboard, finger, voice command, or other assistive technology. <button> </button> Link
Caption Specifies the caption (or title) of a table. <caption> </caption> Link
Cite Used to mark up the title of a cited creative work. <cite> </cite> Link
Code Displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code. <cite> </cite> Link
Col Element defines a column within a table and is used for defining common semantics on all common cells. <col> </col> Link
Colgroup Defines a group of columns within a table. <colgroup> </colgroup> Link
Data Links a given piece of content with a machine-readable translation. <data> </data> Link
Datalist contains a set of <option> elements that represent the permissible or recommended options available to choose from within other controls. <datalist> </datalist> Link
Dd provides the description, definition, or value for the preceding term (<dt>) in a description list (<dl>). <dd> </dd> Link
Del Represents a range of text that has been deleted from a document. This can be used when rendering "track changes" or source code diff information. <del> </del> Link
Details Element creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state. <details> </details> Link
Dfn Element is used to indicate the term being defined within the context of a definition phrase or sentence. <dfn> </dfn> Link
Dialog Represents a dialog box or other interactive component, such as a dismissible alert, inspector, or subwindow. <dialog> </dialog> Link
Div The generic container for flow content. It has no effect on the content or layout until styled in some way using CSS (e.g. styling is directly applied to it, or some kind of layout model like Flexbox is applied to its parent element). <div> </div> Link
Dl represents a description list. The element encloses a list of groups of terms (specified using the <dt> element) and descriptions (provided by <dd> elements). <dl> </dl> Link
Dt Specifies a term in a description or definition list, and as such must be used inside a <dl> element. <dt> </dt> Link
Embed Embeds external content at the specified point in the document. <embed> </embed> Link
Em Marks text that has stress emphasis. <em> </em> Link
Fieldset element is used to group several controls as well as labels (<label>) within a web form. <fieldlist> </fieldlist> Link
Figcaption Represents a caption or legend describing the rest of the contents of its parent <figure> element. <figcaption> </figcaption> Link
Figure Represents a self-contained content, potentially with an optional caption, which is specified using the <figcaption> element. <figure> </figure> Link
Form Represents a document section containing interactive controls for submitting information. <form> </form> Link

Hr Represents a thematic break between paragraph-level elements: for example, a change of scene in a story, or a shift of topic within a section. <hr> </hr> Link
I Represents a range of text that is set off from the normal text for some reason, such as idiomatic text, technical terms, taxonomical designations, among others. <i> </i> Link

Img Embeds an image into the document. <img> </img> Link
Ins Represents a range of text that has been added to a document. <ins> </ins> Link

Label Represents a caption for an item in a user interface. <label> </label> Link
